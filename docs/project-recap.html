Project Identity

citation-tracker v0.1.0

An automated pipeline for tracking academic citations and performing deep qualitative analysis using LLMs. Designed for researchers who want to see not just *who* cited them, but *how* their work is being used, challenged, or extended.

Python 3.12+ SQLite (WAL) OpenRouter / Claude PyMuPDF4LLM

Recent Activity

The Great Refactor & Hardening

A massive push that transitioned the prototype into a professional CLI tool.

  • ID Overhaul: Switched from integers to 8-character short UUIDs for global stability.
  • Local First: Pivoted from email (Resend) to local HTML reports in ~/.citation-tracker/reports.
  • UI/UX: Integrated glow for terminal Markdown and rich for beautiful CLI output.
  • CI/CD: Established GitHub Actions and comprehensive unit test coverage.

Decision Log

Short UUIDs (8-char)

Decided: Use uuid.uuid4().hex[:8] for all entities.

Rationale: Avoids the "leaky abstraction" of sequential IDs and prevents collisions if the database is reset or synced across devices, while staying short enough for easy terminal input.

PyMuPDF-Layout Integration

Decided: Add pymupdf-layout as a core dependency.

Rationale: Standard PDF parsers struggle with multi-column academic layouts. Explicit layout analysis ensures the LLM receives the text in the correct reading order.

Local-Only Reports

Decided: Abandon email delivery in favor of a local reporting directory.

Rationale: Reduces external dependencies (no API keys for mailers) and creates a persistent, searchable archive of research impact on the user's machine.

Mental Model Essentials

  • The 8-Char Key: Everything revolves around the short ID. Use list to find them, and run --id [ID] to focus.
  • Storage: ~/.citation-tracker/ is the source of truth. If things get weird, check the SQLite DB there.
  • Structured Output: The analyser.py requires valid JSON from the LLM. It's the most brittle part of the pipeline.
  • Open Access: The tool is optimized for OA papers via Unpaywall. For paywalled papers, manual ingestion (ingest) is the intended escape hatch.

Cognitive Debt Hotspots

High

Analyser Context Limits

Current chunking logic in analyser.py is simple. Very long papers might lose nuance during summarization before the final relationship analysis.

Suggestion: Implement a "map-reduce" style analysis for complex papers.

Medium

Fetch Reliability

Dependency on Unpaywall means many citations might be found but not "fetched."

Suggestion: Add more aggressive PDF discovery (e.g., ArXiv API, Crossref) or better UI feedback for failed fetches.

Next Steps

  • Network Visualization: Generate Mermaid or D3 graphs showing the "influence tree" of papers.
  • Batch Mode: Improve the run command to handle hundreds of citations more gracefully with better concurrency.
  • Google Scholar: Investigate scrapers or alternative APIs to bridge the gap in citation discovery.
